# 第二阶段框架化提取计划

## 目标

把第一阶段 PoC 整理成可维护的 Appium 自动化框架。

## 计划重点

- driver 创建和 capabilities 管理。
- base page 页面基类和 locator 约定。
- 显式等待工具。
- 通用手势。
- 权限弹窗和普通弹窗处理。
- 诊断产物收集。
- 测试数据管理。
- 环境配置。

## 进入条件

- 第一阶段已经有至少一个稳定生成脚本。
- 生成脚本已经通过重复执行验证。
- 已经理解目标 App 的常见启动流程。

## 退出条件

- 新测试可以复用 driver 和诊断工具。
- 至少存在两个常用页面的 page object。
- 第二条测试用例可以用比第一阶段更少的重复代码创建。
